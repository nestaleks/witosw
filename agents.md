## 8. Адаптивность

Сайт должен быть полностью адаптивным, а не просто уменьшенной десктопной версией.

Обязательно проверь следующие ширины:

- 1440 px;
- 1280 px;
- 1024 px;
- 768 px;
- 480 px;
- 390 px;
- 360 px.

На всех разрешениях:

- не должно быть горизонтальной прокрутки;
- текст не должен выходить за границы;
- элементы не должны перекрываться;
- изображения не должны деформироваться;
- кнопки должны оставаться удобными;
- отступы должны быть визуально сбалансированными;
- контент должен сохранять логичный порядок;
- интерфейс должен оставаться читаемым;
- интерактивные элементы должны иметь удобную область нажатия.

Используй mobile-first или обоснованные медиазапросы.

На планшете и мобильных устройствах допускается перестройка композиции, если буквальное сохранение десктопного расположения ухудшает читаемость. При этом сохрани стиль, содержание, визуальную иерархию и характер дизайна.

## 9. Header и мобильное меню

Создай полноценный адаптивный header.

### На десктопе

- Меню отображается в строку.
- Пункты меню имеют hover-, focus- и active-состояния.
- Активный пункт визуально выделяется.
- Все элементы выровнены и соответствуют референсу.
- Header корректно работает при изменении ширины окна.

### На мобильных устройствах

- Обычная навигация заменяется кнопкой-бургером.
- Меню открывается и закрывается плавно.
- Иконка бургера анимируется в крестик.
- Меню закрывается:
  - при повторном нажатии на кнопку;
  - при выборе пункта меню;
  - при нажатии вне меню;
  - по клавише `Escape`.
- При открытом меню прокрутка страницы блокируется.
- Используются `aria-expanded`, `aria-controls` и `aria-label`.
- Меню доступно с клавиатуры.
- Интерактивные элементы имеют область нажатия не менее примерно `44 × 44 px`.

## 11. Hover-, focus- и tap-эффекты

### Для десктопных устройств

Реализуй аккуратные hover-эффекты для:

- кнопок;
- ссылок;
- карточек;
- изображений;
- пунктов меню;
- интерактивных декоративных элементов.

Допустимые эффекты:

- изменение цвета;
- изменение фона;
- легкое смещение;
- мягкое масштабирование;
- изменение тени;
- анимация линии;
- подчеркивание;
- изменение прозрачности.

Не используй чрезмерные вращения, сильные прыжки и агрессивные эффекты, если их нет в референсе.

### Для клавиатуры

Добавь хорошо заметные `:focus-visible`-состояния.

### Для мобильных устройств

- Не полагайся на `hover`.
- Добавь `:active`, `:focus-visible` и управляемые JavaScript tap-состояния там, где это действительно нужно.
- При тапе пользователь должен получать понятную визуальную обратную связь.
- Tap-эффекты не должны оставаться навсегда.
- Не допускай двойного срабатывания действий.

Используй media queries по возможностям устройства:

```css
@media (hover: hover) and (pointer: fine) {
  /* desktop hover effects */
}

@media (hover: none), (pointer: coarse) {
  /* touch and tap states */
}
```

## 12. Анимации

Добавь современные, но ненавязчивые анимации:

- плавное появление секций при прокрутке;
- небольшое перемещение;
- изменение прозрачности;
- анимацию кнопок;
- раскрытие мобильного меню;
- плавные переходы состояний;
- легкие эффекты для карточек и изображений.

Используй:

- CSS transitions;
- CSS keyframes только там, где они оправданы;
- Intersection Observer для появления элементов при прокрутке.

Требования:

- анимации не должны ухудшать производительность;
- анимируй преимущественно `transform` и `opacity`;
- не создавай постоянных тяжелых анимаций;
- элементы не должны оставаться невидимыми, если JavaScript не загрузился;
- поддержи `prefers-reduced-motion`.

Обязательная поддержка:

```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    scroll-behavior: auto !important;
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```
